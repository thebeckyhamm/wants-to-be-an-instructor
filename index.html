The BECKY HAMM

wants to be a TIY instructor

and here's why

Every user should be able to access content

Whether they are in China on Android 4.3, in Nashville on Google fiber, or in Antigua on DSL, everyone needs to see the same content, and see it in a timely manner. This multi-device world is here to stay.

Clean Markup • Performance • Content Parity • Responsive Design

that being said...

Loading the site is only half the battle

If a visitor can't use the site, why was it made?

Even before wireframing, the overall goal of the project needs to be determined. And every decision made during development should keep that goal in mind.

Purpose • User Experience • Client Interaction

speaking of design...

Design does not equal "looking pretty"

Creating a well-designed site requires more ingredients that just color and type.

Is red the best color for a serious financial institution? Will users get lost if we use a hamburger menu? How will we make the site accessible for non-sighted users? What would Dieter Rams do?

Branding • Accessibility • Design

so how is all of this taught?

By breaking complex concepts down into simple steps

Myriad factors go into designing a site from start to finish. Students should have the right blueprints to guide them from <a> tags to gzip.

And I want to give them that guidance!
